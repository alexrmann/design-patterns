Region Landmarks

Regions can be labeled based on their first child heading using the aria-labelledby attribute on the section element. The label must match the id attribute of the child heading.

This section uses its h2 element with id="regions". Consequently, the section has the attribute aria-labelledby="regions".

Design Patterns

  • A region landmark must have a label.
  • If a page includes more than one region landmark, each should have a unique label.
  • The region landmark can be used identify content that named landmarks do not appropriately describe.

Complementary Landmarks

The HTML aside element represents a portion of a document whose content is only indirectly related to the documentâ€™s main content. Given its indirect relationship to main, it should not be contained within the main content.

Design Patterns

  • complementary landmarks should be top level landmarks (e.g. not contained within any other landmarks).
  • If the complementary content is not related to the main content, a more general role should be assigned (e.g. region).
  • If a page includes more than one complementary landmark, each should have a unique label.

The HTML footer element defines a contentinfo landmark when its context is the body element.

Contentinfo Landmark

The HTML footer element defines a contentinfo landmark by default when its context is the body element.

The HTML footer element is not considered a contentinfo landmark when it is descendant of any of following elements:

  • article
  • aside
  • main
  • nav
  • section
